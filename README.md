# 🧩 LeetCode Solutions

A personal LeetCode problem-solving journal focused on **optimization, code quality, and iterative improvement**, written in **Java.**

Each solution includes:

- **Intuition & Approach** — The initial idea and strategy after understanding the problem
- **Initial Solution** — The first working implementation

- **Complexity Analysis**
    - **Time Complexity** — Runtime analysis
    - **Space Complexity** — Memory usage analysis

- **Review** — Identification of inefficiencies and bottlenecks
- **Revised Solution** — Optimized implementation with improvements explained
- **Performance Comparison** — Runtime and memory comparison before vs. after optimization

## 🔍 Problem-Solving Philosophy

- Start with a working solution, then iteratively optimize
- Small implementation details (e.g., data structures) can significantly impact performance
- Prioritize both readability and efficiency — well-structured code often leads to better performance
