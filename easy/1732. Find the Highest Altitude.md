# [1732. Find the Highest Altitude](https://leetcode.com/problems/find-the-highest-altitude/description/?envType=study-plan-v2&envId=leetcode-75)

## Intuition

The biker starts at an altitude of 0.<br>
By cumulatively adding each value in the `gain` array, we can determine the current altitude at each step.

The goal is to track the **maximum altitude reached during this cumulative process.**

---

## Approach

1. Initialize a variable `altitude` to 0 to track the current altitude.
2. Initialize a variable `max` to 0 to track the highest altitude.
3. Iterate through the gain array:
   - `altitude += gain[i]`
   - `max = Math.max(max, altitude)`
4. Return `max`.

---

## Complexity
### Time Complexity
O(n), where n is the length of the array

### Space Complexity
O(1), as only a few variables are used regardless of input size

---

## My Solution
```
class Solution {
    public int largestAltitude(int[] gain) {
        int[] altitude = new int[gain.length + 1];
        altitude[0] = 0;

        for (int i = 0; i < gain.length; i++) {
            altitude[i + 1] = altitude[i] + gain[i];
        }

        int max = -1;
        for (int n : altitude) {
            if (max < n) max = n;
        }

        return max;
    }
}
```

---

## Review

The runtime is already optimal at 0ms, but there is room for improvement in terms of space efficiency.<br>
The current implementation uses an `altitude` array, which requires O(n) additional space. However, only the previous altitude value is needed to compute the next one.

Therefore, we can eliminate the array and directly maintain a running sum to reduce space usage.

- Remove the `altitude[]` array → Space Complexity: O(n) → O(1)
- Combine two loops into a single loop to eliminate unnecessary iteration
- Since the starting altitude is always 0, initializing `max = 0` is safer

---

## Revised Solution
```
class Solution {
    public int largestAltitude(int[] gain) {
        int max = 0;
        int altitude = 0;
        
        for (int g : gain) {
            altitude += g;
            if (max < altitude) max = altitude;
        }

        return max;
    }
}
```

### Comparison Summary

| Category | Before | After |
|---|---|---|
| Altitude tracking | Uses `int[]` array (size n+1) | Uses single `int` variable |
| Loop count | 2 separate loops | 1 combined loop |
| Max initialization | `-1` (potential bug) | `0` (correct: start altitude) |
| Time Complexity | O(n) | O(n) |
| Space Complexity | O(n) | O(1) |

This improvement is not a change in the algorithm itself,<br>
but rather an **implementation refinement focused on eliminating unnecessary array allocation.**

### Key Improvements
- **Array Elimination**
  - Removed `int[] altitude` array of size `n+1`
  - Replaced with a single rolling `int altitude` variable
- **Runtime Efficiency**
  - Runtime maintained at **0ms**, percentile ranking held at **100.00%.**
  - → No regression; the optimization did not trade time for space.
- **Memory Efficiency**
  - Absolute memory usage dropped from **43.29MB to 42.92MB**,
  - and the percentile ranking improved significantly from **19.27% to 73.98%.**
  - → Eliminating the altitude array removes heap allocation proportional to input size, reducing GC pressure.
- **Loop Consolidation**
  - Before: two passes — one to build the altitude array, one to find the max.
  - After: single pass — altitude and max updated together per iteration.
  - → Reduces loop overhead and improves cache locality by avoiding array traversal.
- **Bug Fix**
  - Before: `max = -1` could theoretically mishandle edge cases where the start altitude (0) is the true maximum.
  - After: `max = 0` correctly reflects that altitude begins at 0.
