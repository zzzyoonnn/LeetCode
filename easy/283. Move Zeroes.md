# [283. Move Zeroes](https://leetcode.com/problems/move-zeroes/description/?envType=study-plan-v2&envId=leetcode-75)

## Intuition
We need to move all zeros to the end of the array while maintaining the **relative order** of the non-zero elements.<br>
A straightforward approach is to swap a zero with the next non-zero element, but this leads to unnecessary repeated scans and is inefficient.

To improve this,
we can **shift all non-zero elements forward and then fill the remaining positions with zeros.**

---

## Approach
1. Iterate through the array and find a zero.
2. When a zero is found, search for the next non-zero element and swap them.

---

## Complexity
### Time Complexity
O(n²), where n is the length of the array

### Space Complexity
O(1), no additional space used (in-place swapping)

---

## My Solution
```
class Solution {
    public void moveZeroes(int[] nums) {
        for (int zero = 0; zero < nums.length - 1; zero++) {
            if (nums[zero] != 0) continue;

            for (int number = zero + 1; number < nums.length; number++) {
                if (nums[number] == 0) continue;

                int temp = nums[zero];
                nums[zero] = nums[number];
                nums[number] = temp;
                break;
            }
        }

        System.out.println(Arrays.toString(nums));
    }
}
```

---

## Review
Due to the nested loop structure, unnecessary iterations are repeated,<br>
and performance degrades significantly, especially when there are many zeros.

The key to this problem is not “swapping,” but<br>
**minimizing unnecessary movements.**

---

## Revised Solution
```
class Solution {
    public void moveZeroes(int[] nums) {
        int zero = 0;
        
        for (int num : nums) {
            if (num != 0) {
                nums[zero++] = num;
            }
        }

        while (zero < nums.length) {
            nums[zero++] = 0;
        }
    }
}
```

### Comparison Summary

| | Before | After |
|---|---|---|
| Time Complexity | O(n²) | O(n) |
| Space Complexity | O(1) | O(1) |
| Approach | Swap | Overwrite + Fill |

### Key Improvements
- **Reduced Iterations**
  - Double loop → Single loop
  - Eliminated redundant iterations
- **Runtime Efficiency**
  - Runtime dropped from 55ms to 1ms,
  - and the percentile ranking improved significantly from **5.41% to 99.85%.**
  - → Marking visited nodes at insertion time prevents duplicate queue entries.
- **Memory Efficiency**
  - Absolute memory usage dropped from 48.43MB to 47.55MB,
  - and the percentile ranking improved significantly from **7.55% to 87.83%.**
  - → Prevents the same node from being processed multiple times, reducing GC overhead.
- **Simplified Traversal Logic**
  - Before: Check visited status after dequeuing, allowing duplicate insertions.
  - After: Mark visited immediately when adding to queue.
  - → Improves both readability and maintainability.
