# [345. Reverse Vowels of a String](https://leetcode.com/problems/reverse-vowels-of-a-string/description/?envType=study-plan-v2&envId=leetcode-75)

## Intuition
We need to return a string where the positions of characters remain the same, but **the order of vowels is reversed.**<br>
To achieve this, we first collect all vowels into a list while iterating through the string.<br>
Then, we traverse the string again and replace each vowel with elements from the list in reverse order.

---

## Approach
1. Create an `ArrayList<String>` to store vowels.
2. Traverse the string `s` and add each vowel to the list.
3. Create a `StringBuilder` to build the result string.
4. Traverse the string again:
   - If the current character is a vowel, append the last element from the list.
   - Otherwise, append the character as it is.
5. Convert the `StringBuilder` to a string and return it.

---

## Complexity
### Time Complexity
O(n), where n is the length of the string

### Space Complexity
O(n), additional space used for the vowel list and result string

---

## My Solution
```
import java.util.*;

class Solution {
    public String reverseVowels(String s) {
        ArrayList<String> list = new ArrayList<>();

        for (int i = 0; i < s.length(); i++) {
            char c = s.charAt(i);

            if (isVowels(c)) list.add(String.valueOf(c));
        }

        int index = list.size() - 1;
        StringBuilder sb = new StringBuilder();
        for (int i = 0; i < s.length(); i++) {
            char c = s.charAt(i);

            if (isVowels(c)) {
                sb.append(list.get(index));
                index--;
            } else {
                sb.append(String.valueOf(c));
            }
        }

        return sb.toString();
    }

    public boolean isVowels(char c) {
        return c == 'a' || c == 'e' || c == 'i' || c == 'o' || c == 'u' ||
                c == 'A' || c == 'E' || c == 'I' || c == 'O' || c == 'U';
    }
}
```

---

## Review
This problem can be solved **more efficiently using the two-pointer approach.**<br>
The initial solution collected vowels into a separate list before reconstructing the string, which introduced unnecessary memory usage and object creation.

- `ArrayList<String>` → creates unnecessary objects by converting `char` to `String`
- Reading the list in reverse → can be replaced with two pointers
- Collecting vowels then re-inserting in reverse → **2-pass structure**
- Explicit vowel checks → can be simplified using `String.indexOf()` or a `Set`
- Using a collection for a simple swap operation → inefficient

<br>

The key idea is:<br>
**Move two pointers from both ends, find vowels, and swap them directly.**

With the two-pointer approach:

- No additional collection is required
- The problem can be solved in a single pass
- Memory usage is minimized

---

## Revised Solution
```
class Solution {
    public String reverseVowels(String s) {
        char[] arr = s.toCharArray();
        int left = 0, right = arr.length - 1;

        while(left < right) {
            while(left < right && !isVowel(arr[left])) left++;
            while(left < right && !isVowel(arr[right])) right--;

            if (left < right) {
                char temp = arr[left];
                arr[left++] = arr[right];
                arr[right--] = temp;
            }
        }

        return new String(arr);
    }

    private boolean isVowel(char c) {
        return "aeiouAEIOU".indexOf(c) != -1;
    }
}
```

| Item | Before | After |
|------|--------|-------|
| Data structure | `ArrayList<String>` (object creation) | `char[]` (primitive) |
| Algorithm | 2-pass (collect → re-insert in reverse) | Two-pointer, 1-pass |
| Space complexity | O(n) list + O(n) StringBuilder | O(n) char array only |
| Vowel check | 10 chained conditions | `"aeiouAEIOU".indexOf(c)` |

### Key Improvements
- **Runtime Improvement**
  - Execution time decreased from 7ms to 3ms,
  - and the percentile ranking jumped from **14.49% to 79.93%.**
  - → The two-pointer approach is meaningfully faster than the 2-pass collection method. 
- **Memory Efficiency**
  - Absolute memory usage dropped from 49.10MB to 46.42MB,
  - and the percentile ranking improved significantly from **5.19% to 81.93%.**
  - → Replacing `ArrayList<String>` with `char[]` eliminates unnecessary object creation. 
- **Simplified Data Structure**
  - Before: Collect vowels into an `ArrayList<String>`, then re-insert in reverse.
  - After: Directly swap in-place within a `char[]` using two pointers.
  - → Removing the intermediate collection reduces memory allocation and GC overhead. 
- **Cleaner Vowel Check**
  - Removed 10 chained conditional expressions.
  - Replaced with a single `"aeiouAEIOU".indexOf(c)` call.
  - → Improves both readability and maintainability.
