# [392. Is Subsequence](https://leetcode.com/problems/is-subsequence/?envType=study-plan-v2&envId=leetcode-75)

## Intuition
To determine whether string `s` is a subsequence of `t`,<br>
we need to check if the characters of `s` appear in `t` in **the same order.**

In the initial approach, both strings are converted into lists.<br>
While iterating through `t`, if a character matches the first element of `s`,<br>
that element is removed from the list.

After the iteration, if the list for `s` becomes empty,<br>
it means all characters were found in order, so we return `true`.

---

## Approach
1. If `s` is empty, return `true`.
2. If `t` is empty, return `false`.
3. Convert `s` into a list of characters.
4. Convert `t` into a list of characters.
5. Iterate through `tList`, and if the current character matches the first element of `sList`, remove it.
6. After the loop, return `true` if `sList` is empty; otherwise return `false`.

---

## Complexity
### Time Complexity
O(n × m), where `n` and `m` are the lengths of `s` and `t`, due to repeated `remove(0)` operations

### Space Complexity
O(n + m), additional space used for storing both strings as lists

---

## My Solution
```
class Solution {
    public boolean isSubsequence(String s, String t) {
        if (s.isEmpty()) return true;
        if (t.isEmpty()) return false;
        
        List<String> sList = new ArrayList<>(Arrays.asList(s.split("")));
        List<String> tList = new ArrayList<>(Arrays.asList(t.split("")));

        for (String c : tList) {
            if (!sList.isEmpty() && sList.get(0).equals(c)) sList.remove(0); 
        }

        if (sList.size() > 0) return false;
        else return true;
    }
}
```

---

## Review
This problem can be solved more efficiently using the **Two Pointer** technique.

```
s = "ace"
t = "abcde"

     a b c d e
     ↑
a c e
↑
```

- Avoid unnecessary `String → List` conversion
- `split("")` introduces extra overhead
- `ArrayList.remove(0)` is O(n), which significantly slows down performance

---

## Revised Solution
```
class Solution {
    public boolean isSubsequence(String s, String t) {
        int si = 0, ti = 0;

        while (si < s.length() && ti < t.length()) {
            if (s.charAt(si) == t.charAt(ti)) si++;
            ti++;
        }

        return si == s.length();
    }
}
```

### Comparison Summary

| Category | Original | Optimized |
|----------|----------|----------|
| Data Structure | `List<String>` | Index pointers |
| Time Complexity | O(n²) | **O(n)** |
| Space Complexity | O(n + m) | **O(1)** |
| Core Operation | remove(0) | Simple comparison |

This improvement is not a change in the algorithm itself,

### Key Improvements
- **Removed Unnecessary Conversions**
  - String → List conversion eliminated
  - No more `split("")` overhead or ArrayList allocation
- **Runtime Efficiency**
  - Runtime dropped from 13ms to 2ms,
  - and the percentile ranking improved significantly from **2.46% to 71.21%.**
  - → Replacing `ArrayList.remove(0)` (O(n)) with a simple index pointer (O(1)) eliminated the bottleneck.
- **Memory Efficiency**
  - Absolute memory usage dropped from 44.40MB to 42.66MB,
  - and the percentile ranking improved significantly from **5.81% to 80.40%.**
  - → Removing two List allocations reduced heap usage and GC pressure.
- **Simplified Traversal Logic**
  - Before: Convert both strings to `List<String>`, mutate the list during iteration.
  - After: Use two integer pointers (`si`, `ti`) directly on the original strings.
  - → Improves both readability and maintainability.
